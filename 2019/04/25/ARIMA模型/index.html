ARIMA模型

发表于 2019-04-25 | 分类于 机器学习

       本示例基于 python 3.7.1实现，其中涉及到的ARIMA的原理及理论方法不在本篇中讲解。本篇仅将ARIMA方法预测时间序列的步骤进行讲解，所呈现的最终效果皆为最终效果，省略了中间探索的步骤。因此，在建立ARIMA模型过程中的探索步骤，希望读者自己进行。本示例采用的数据集为2016年的风功率，读者可以访问以下连接进行下载。另外，在此提供一个开源的数据集的网站（UCI，University of California Irvine的机器学习库：数据集http://archive.ics.uci.edu/ml/datasets.php），读者可自行下载自己感兴趣的数据集，做相关数据分析。

自相关和偏相关

       为了更清楚的看到风功率变化的时间序列的特点，我们首先计算其自相关和偏相关。

可以看到结果如下

       自相关和偏相关函数图像可以反映很多问题，可以根据函数图像来判断时间序列的平稳性。平稳过程的自相关系数和偏自相关系数都会以某种方式衰减趋于0，前者测度当前序列与先前序列之间简单和常规的相关程度，后者是在控制其它先前序列的影响后，测度当前序列与某一先前序列之间的相关程度。如果某一时间序列的自相关函数随着滞后k的增加而很快地下降为0，那么我们就认为该序列为平稳序列；如果自相关函数不随着k的增加而迅速下降为0，就表明该序列不平稳。如果一个时间序列的自相关和偏相关图没有任何模式，而且数值很小，那么该序列可能就是一些互相独立的无关的随机变量。
       可以从图中明显的看到，自相关函数随滞后数的增加有下降的趋势，但并没有迅速的变为0，而偏相关函数值在滞后数有拖尾现象。初步判断数据需要进行平稳化处理。

ADF检验

       接下来，我们使用ADF检验，来检验时间序列的平稳性。

  • adfuller(timeseries, autolag=’AIC’) autolag: 按某个度量自动选择测试使用的lag数量以使该度量最小化
  • AIC: Akaike Information Criterion, 估算两个回归模型之间的KL散度/相对熵; KL散度/相对熵用于衡量近似分布与真实分布的接近程度

       在进行移动平均的时候，移动窗口数目的选择十分重要，这会影响到尖峰值的平滑程度，或者说会影响到移除的信息。
       显然经过ADF检验以后，我们已经明确了原时间序列的不平稳的特性，需要对原序列进行平稳化处理。

ARIMA模型的建立

       ARIMA模型含有三个参数，即ARIMA(p,d,q)，在这里简单的说明三个参数的设立原则。

  • d：差分阶数，即使序列达到平稳的差分阶数。序列在lag>1出有显著的正自相关, 需提高差分阶数；lag=1 时自相关为0或负值, 无需差分；lag=1 时自相关< -0.5, 则为过差分
  • p: 如果PACF(lag<=k)显著且PACF(lag>k)均不显著, 考虑将p设为k；如果差分后序列的PACF迅速衰减到0而ACF在lag较高处仍有显著的较强的相关, 和/或 ACF(lag=1) > 0, 考虑增加p
  • q: 如果ACF(lag<=k)显著且ACF(lag>k)均不显著, 考虑将q设为k；如果差分后序列的ACF迅速衰减到0, 和/或 ACF(lag=1) < 0, 考虑增加q
           以上方式仅为一个参考，当然也可以通过遍历的方式，选择出最佳的值，依据准则一般为BIC或AIC。
           我们选择模型ARIMA(1,1,1)，参数的选择是否有更优方案，读者可以自己尝试。在建立模型前我们首先将数据集拆分为训练集和测试集，本示例以2016年前11个月为训练集，12月数据为测试集。

       在切分数据集之后，开始进行模型训练，model.fit()为训练，model.forecast()为预测。在这里，我们通过历史数据预测下一个值，预测完毕后，将真值纳入历史数据，继续预测下一个值。注意，笔者在纳入旧值的时候剔除新值，有兴趣的读者可以下来自行实现。
